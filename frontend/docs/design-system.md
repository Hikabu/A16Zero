# Spacing System

Use Tailwind spacing scale only.

Preferred spacing:
- 2
- 4
- 6
- 8
- 12
- 16
- 24

Avoid arbitrary spacing values.

---

# Radius System

Small: rounded-lg
Default: rounded-xl
Cards: rounded-2xl

---

# Shadows

Use subtle shadows only.

Cards:
- soft shadow
- low opacity
- minimal blur

Avoid heavy floating shadows.

---

# Layout Rules

## Max Widths

Dashboard:
- max-w-7xl

Forms:
- max-w-xl

Reading Content:
- max-w-4xl

---

# Card System

## Default Cards

Cards should:
- use subtle borders
- use soft surfaces
- maintain strong padding consistency
- avoid excessive visual nesting

Default:
- rounded-2xl
- border
- bg-surface
- p-6

# Input System

Inputs should:
- feel dense but readable
- use subtle borders
- have visible focus states
- avoid bright glows

Default:
- dark surface
- border
- rounded-xl
- h-11 minimum

---

# Tables

Tables should:
- prioritize scanability
- use row separation
- avoid visual clutter
- support sorting/filtering

Avoid giant dense spreadsheets.

---

# Metric Components

Metrics should:
- emphasize readability
- clearly separate labels and values
- support confidence states

Use:
- progress bars
- percentage indicators
- confidence badges

---

# Confidence Indicators

Confidence is a core product concept.

## High Confidence
- green accent
- strong signal

## Medium Confidence
- yellow accent
- neutral signal

## Low Confidence
- muted accent
- uncertainty visible

Confidence should NEVER feel hidden.

---

# Loading States

Use:
- skeleton loaders
- staged progress indicators
- queue/progress messaging

Avoid:
- giant spinners
- blank pages

---

# Empty States

Empty states should:
- explain why data is missing
- explain next action
- maintain product trust

Tone:
- calm
- technical
- informative

---

# Charts

Charts should:
- be minimal
- support quick understanding
- avoid rainbow colors
- avoid unnecessary complexity

Preferred:
- bar charts
- trend charts
- radar charts sparingly

---

# Animation Rules

Motion should:
- support understanding
- remain subtle
- feel fast

Avoid:
- bouncy animations
- long transitions
- distracting effects

---

# Mobile Rules

Mobile should:
- prioritize vertical flow
- collapse dense data intelligently
- preserve readability
- maintain spacing consistency

---

# Visual Identity Summary

The UI should feel like:
- a trusted hiring intelligence system
- a premium technical platform
- a modern recruiter workflow product
- an evidence engine
