3. CANDIDATE CREATES TECHNICAL SCORECARD:
3 modes : GITHUB | WALLET | GITHUB + WALLET
> must sync first regardless of auth methods

3.1 SYNC ACCOUNTS

3.1.1 SYNC GITHUB > 
Required in order to get scorecard mode GITHUB | GITHUB + WALLET
GET
/sync/github/connect
Start GitHub connection > required step 1 regardless of auth
> if correct redirects to http://localhost:3001/dashboard/github/syncing
> must update in frontend for a redirect after github is successfully connected


POST
/sync/github
Sync Github before scorecard generation > here we fetch all data, a successfull /sync/github/connect is required


GET
/sync/github/status
Get GitHub sync status > sync/github is usually inmediate but you can check the status here in case it takes some time


3.1.2 SYNC WALLET >  
Required in order to get scorecard mode WALLET | GITHUB + WALLET

GET
/sync/wallet/challenge
Generate wallet linking challenge
> Returns a "message"
> Frontend must ask user to sign this message with their wallet and record signature


POST
/sync/wallet
Verify wallet signature and link wallet
> Frontend must send the previous signature to this endpoint to finalize sync

3.2 COMPUTE SCORECARD 
Proof Of Talent

POST
/api/analysis
Trigger developer analysis
> Can be used for users and non users
> For users signed in send no body
> For non registered users can be used with a body (no verification of ownership) > this mode can be used in order to get potential candidates to use the product before signing update


POST
/api/analysis/recompute
Recompute analysis
> ANalysis are rate limiited to 1 per 24 hours. To override this we can use an internal call admin. 
>> This is more for testing no need to implement this in the frontend flow.
>> Can only be used for a registered user with their id... 


GET
/api/analysis/{jobId}/status
Get job status
The scorecard analysis is pretty fast but you can check and update status in frotnend with this endpoint to check the current stage of analyis

GET
/api/analysis/{jobId}/result
Get job result
When the score card is done there will be no ntoifications to the frontend. 
>> For now the forntend can poll status and result endpoint to see the result.
